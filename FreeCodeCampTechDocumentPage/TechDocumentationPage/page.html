Introduction

Python is a cross-platform, object-oriented, dynamic language.Designed for ease of use and to be readable even to the beginning programmer, Python is a perfect language for any novice programmer.

Don't think however that because this is so, that you cannot make some impressive things with Python; on the contrary, Python is a powerful language that can be used for just about any large-scale programming project.

Installing Python is very easy to do, and many UNIX & Linux distributions come with Python as an initial package. Sometimes, even some Windows computers already come with Python installed. If you do need to install it, however, feel free to follow this link to do so.

The Basics of Python

One of the most important things to know about Python is that everything in Python is an object, derived from one of Python's built-in Types.

Python's built-in types are:

  • Numerics
  • Sequences
  • Mappings
  • Classes
  • Instances
  • Exceptions

Numerics

Python has three distinct numeric types:

  • int - integers
  • float - floating point numbers
  • complex - complex numbers

Numerics are used for to store numbers, whether coming from a function or something you might need later on. You can do everything from simple to complex math using Numerics.

Like most things in Python, they are easy to create:

a = 5

type(a)

>>> class 'int'

The above code creates an "a" variable, and because we placed the integer 5 in it, it is now a instance of the "int" class.

Sequences

Python has three basic sequence types:

  • list - list
  • tuple - tuple
  • range - range

Sequences are the generic term for an ordered set. The three listed above change in mutability, but like are great for storing all kinds of data.

Lists

Lists are the most versatile sequence type. Lists are mutable, and any object can be put in one. Elements within can be reassigned, removed or inserted.

Tuples

Tuples are very similar to Lists, but they are immutable; once made, they cannot be changed. This makes them perfect for data you want stored but don't want to put at risk of being changed by code.

Strings

These are special type of sequence that can only store characters. Most use them to store words that you might want printed to the user as well.

Mappings

Mappings are mutable objects that map hashable values to arbitrary objects.

Python actually only has one mapping object:

  • dict - dictionaries
Dictionaries

Dictonaries, much like their namesake, consist of key and value pairs. If a dictionary key is called, it will return its value. Likewise, if a dictionary value is called, it will return whatever key is associated with it.

a = {'first':15, 'second':30}

a.keys()

>>> dict_keys(['first', 'second'])

a.values()

>>> dict_values([15, 30])

Classes

A class is best described as a code template for creating objects. They are created by using the constructor of the class. Objects called this way are called an instance of the class.

If you'll remember, everything in Python is an object. This means they also belong to a class and are thus instances of those classes.

Classes also have special operations called methods, which are functions that only instances of the class they are associated with can use.

Classes allow you to use a form of programming called Object Oriented Programming, which opens up a brand new way for you to make programs from the traditional functional programming.

Make a class

class Snake:

name = "python"

Create a class instance

snake = Snake()

Access class attribute name inside class

print(snake.name)

>>> python

Instances

As described in Classes, Instances are an individual object of a certain class. An object that belongs to a class Person, for example, is an instance of the class Person, with all the methods and characteristics that belong to all instances of that class.

Exceptions

An exception is an event which occurs during the execution of a program that disrupts the normal flow of the program's instructions. When such an event occurs and a Python script encounters a situation that it cannot cope with, it raises an exception. An Exception then is a Python object that represents an error.

Common exceptions in Python include:

  • ImportError
  • NameError
  • SyntaxError
  • TypeError
  • PermissionError
